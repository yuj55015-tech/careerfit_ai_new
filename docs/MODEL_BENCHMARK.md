## 1. 문서 목적



이 문서는 CareerFit AI 프로젝트에서 사용할 AI 모델을 선택하기 위해 작성한다.



공개 리더보드 점수만 보고 모델을 고르지 않고,

내 프로젝트의 실제 사용 목적에 맞게 직접 질문하고 비교한다.



CareerFit AI의 모델 선택 기준은 다음과 같다.



* 한국어 답변 품질

* 취업·직무 상담 적합성

* 채용공고 요약 능력

* 사용자 맥락 반영 능력

* 환각이 적은 답변

* 응답 속도

* 비용 또는 무료 사용 가능성

* API 연결 난이도

* 라이선스와 사용 제한

## 3. 모델 후보 선정 기준



모델을 고를 때는 아래 항목을 먼저 확인한다.



### 3-1. 기본 확인 항목



* [ ] 내가 하려는 작업과 모델의 Task가 일치하는가?

* [ ] API로 호출 가능한 모델인가?

* [ ] 무료 티어 또는 테스트 가능한 크레딧이 있는가?

* [ ] 라이선스가 교육/포트폴리오/서비스 목적에 맞는가?

* [ ] 한국어 입력과 출력이 가능한가?

* [ ] 모델 카드에 사용법과 제한사항이 적혀 있는가?

* [ ] 최근에도 사용되거나 업데이트된 모델인가?

* [ ] 응답 속도가 실습에 적절한가?

* [ ] 비용 구조가 명확한가?



### 3-2. 라이선스 확인 항목



* [ ] License가 `apache-2.0`, `mit`, `bsd-3-clause` 등 비교적 사용하기 쉬운 라이선스인가?

* [ ] `Non-commercial`, `Research only`, `No commercial use` 문구가 있는가?

* [ ] Gated model이라 별도 승인이나 약관 동의가 필요한가?

* [ ] README 또는 Model Card에 추가 제한사항이 있는가?



### 3-3. 비용 확인 항목



* [ ] Hugging Face Inference Providers로 호출 가능한가?

* [ ] Inference Providers 가격 또는 Billing 차감 기준을 확인했는가?

* [ ] 직접 Inference Endpoint 배포가 필요한 모델인가?

* [ ] 무료 크레딧 초과 시 과금 가능성이 있는가?

* [ ] 아주 짧은 요청으로 1회 테스트 후 Usage를 확인했는가?

## 4. 평가 방식



모델 평가는 1차 정량 평가와 2차 정성 평가로 나눈다.



### 4-1. 정량 평가



각 항목을 1점부터 5점까지 평가한다.



| 점수 | 의미 |

| -- | -------- |

| 5점 | 매우 좋음 |

| 4점 | 좋음 |

| 3점 | 보통 |

| 2점 | 부족함 |

| 1점 | 사용하기 어려움 |



### 4-2. 정성 평가



점수만으로 판단하지 않고, 실제 답변을 읽고 아래 내용을 기록한다.



* 답변이 자연스러운가?

* 사용자의 전공, 경험, 목표를 반영했는가?

* 없는 정보를 지어내지 않았는가?

* 너무 일반적인 조언만 하지 않았는가?

* 실제 포트폴리오 개선에 도움이 되는가?

* 답변 형식이 보기 쉬운가?

* 한국어 표현이 어색하지 않은가?

| 평가 항목 | 설명 | 점수 |

| 정확성 | 질문 의도에 맞게 답했는가 | 1~5 |

| 직무 적합성 | 취업·직무·포트폴리오 맥락을 잘 반영했는가 | 1~5 |

| 맥락 반영 | 사용자 전공, 경력, 관심사를 반영했는가 | 1~5 |

| 환각 방지 | 없는 정보를 지어내지 않았는가 | 1~5 |

| 답변 구조화 | 표, 목록, 단계 등으로 보기 쉽게 정리했는가 | 1~5 |

| 실행 가능성 | 사용자가 바로 실행할 수 있는 조언인가 | 1~5 |

| 응답 속도 | 실습 또는 서비스에서 기다릴 만한 속도인가 | 1~5 |

| 비용 효율 | 무료/저비용 실습에 적합한가 | 1~5 |

| API 연결 난이도 | 프로젝트 코드에 연결하기 쉬운가 | 1~5 |


## 9. 모델 선택 기준



CareerFit AI에서는 모델을 하나만 고정하지 않고, 상황에 따라 다르게 사용할 수 있다.



### 9-1. 기본 답변 생성 모델



사용자 질문에 대한 최종 답변 생성에 사용한다.



선택 기준:



* 한국어 답변이 자연스러움

* 직무 상담 맥락을 잘 이해함

* 답변 구조화가 좋음

* API 연결이 안정적임



후보:



```text

Gemini 2.5 Flash-Lite

Hugging Face Instruct 모델

```



### 9-2. 무료 실습용 모델



수업 중 API 한도 소진을 줄이기 위해 사용한다.



선택 기준:



* 무료 또는 저비용 테스트 가능

* 짧은 프롬프트에서 안정적으로 응답

* 설치와 연결이 간단함



후보:



```text

Hugging Face Inference Providers 지원 모델

Ollama 로컬 모델

Mock Response

```



### 9-3. 한도 초과 대비 모델



API 비용이나 한도 문제가 생겼을 때 사용한다.



선택 기준:



* API Key 없이 동작 가능

* 응답 형식이 일정함

* 수업 흐름을 끊지 않음



후보:



```text

Mock Response

Ollama local model

```
